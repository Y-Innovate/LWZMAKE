What is make?

Make is a utility originally from the unix/linux world used primarily for build automation. In general terms make can automatically update some files from others whenever the others change. The utility takes a script as input, written in make's own script language, called a makefile. In such a script you define build targets, their dependencies and the recipes to build the targets.

A basic example of the makefile syntax: 

helloworld.o: helloworld.c
	gcc -c -o helloword.o helloworld.c

Or in a more generic form: 

target: component
<tab>command

On the first sample line before the : is specified the file or files that share the same command line(s) to build them. In the example that's helloworld.o. After the : follow prerequisites to the building of this target, for example input files, which in turn could be targets defined elsewhere in the makefile.
The second line in the example defines the recipe, in this example a single line, but a recipe can contain multiple lines that each start with a recipe prefix, the default is a tab character. Each recipe line is a command line that is executed to build the target.
